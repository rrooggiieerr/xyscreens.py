## Class Structure in `xyscreens.py`

- **`XYScreensCommands`** — Builds 3-byte RS-485 command frames (`0xFF` + address + cmd)
- **`XYScreensState(IntEnum)`** — `STOPPED=0, UP=1, UPWARD=2, DOWNWARD=3, DOWN=4`
- **`XYScreensConnectionError`** — Raised on serial/TCP connection failures
- **`XYScreens`** — Main class. Key patterns:
  - Constructor accepts endpoint string; `create_tcp()` / `create_serial()` class methods available
  - `_is_tcp_connection()` detects connection type by string format
  - Every command has sync + async variants (e.g., `up()` / `async_up()`)
  - `_post_up()`, `_post_stop()`, `_post_down()` update state after command send
  - `update_status()` recalculates position from `time.time_ns()` deltas and configured durations
  - `async_set_position()` spawns a background task via `asyncio.create_task()`
  - Callbacks notify listeners on state changes

## Adding a New Command

1. Add command byte constant to `XYScreensCommands` (e.g., `_NEW = b"\xNN"`)
2. Add method to `XYScreensCommands` returning the 3-byte frame
3. Add sync method to `XYScreens` using `self._send_command()`
4. Add async method using `self._async_send_command()`
5. Add tests in `tests/test_xyscreens.py`
